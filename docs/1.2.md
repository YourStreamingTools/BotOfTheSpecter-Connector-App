# Version 1.2 - unreleased notes - unknown date for now

## Overview

BotOfTheSpecter v1.2 continues the effort to make things clearer and safer for streamers: a modern **dark-mode UI**, responsive Channel Points editor with thumbnails, and a set of reliability fixes (non-blocking OBS disconnect, debounced requests, and clearer logging). These updates were driven directly by user feedback to improve usability, reduce surprises (freezes or repeated network calls), and make Channel Points management faster and more visually helpful.

**This release focuses on UX polish and operational reliability while keeping backward-compatible behavior.**

---

## What's New ✨

### Modern Dark UI & Layout

- **Left sidebar + stacked pages**: Replaced the old tab bar with a persistent left navigation (Connection / Controls / Variables / Channel Points / Event Log) for faster workflow switching.
- **Consistent dark theme**: Centralized theme (`theme.py`) applied at startup for consistent styling across controls and pages.
- **Header & status improvements**: Clean version label and a compact lock button in the header.

### Channel Points — Responsive & Visual

- **Responsive reward grid**: The Channel Points page now scales to the window size — card columns reflow and resize automatically so the page makes good use of available width.
- **Thumbnails + lazy loading**: Rewards now display thumbnails when available. Images are fetched in the background and displayed as they arrive; a color tile remains as a fallback.
- **In-memory thumbnail caching**: Prevents repeated downloads during a session and speeds UI refreshes.
- **Debounced refreshes**: Multiple triggers (startup, navigation, saving API key, manual Sync) are coalesced into a single network call to avoid hammering the Twitch API.
- **Loading behavior**: Channel Points will load once on application launch. Use the **"Sync from Twitch"** button on the Channel Points page to refresh rewards and thumbnails on demand (manual sync).
- **Click-to-edit reward cards**: Reward cards are now clickable (pointer cursor added) — clicking a card opens the **Reward Editor** for quick edits; the edit button remains available for users who prefer it.
- **Reward Editor — improved UX (no forced scrolling)**: The Reward Editor now auto-sizes to fit available screen space and is resizable (size grip enabled). On most desktop screens users will not need to scroll to see all fields; when screen space is limited the dialog remains resizable.
- **Expanded edit fields & validation**: The editor exposes the full set of Twitch PATCH-able fields (title, prompt, cost, background_color, is_enabled, is_user_input_required, is_paused, should_redemptions_skip_request_queue, per-stream limits, per-user limits, and global cooldown). Client-side validation enforces Twitch constraints (title length, prompt length, numeric ranges) to reduce 400 responses from the Twitch API.

### Variables & Specter integration

- **Default event variables**: The Variables page now shows sensible defaults for common event variables (e.g., `last_follower`, `last_cheer_user`, `last_cheer_amount`, `last_subscriber`, `last_raider`) so the UI displays them even before any events occur.
- **Persisted Specter notices**: Incoming Specter websocket notices are recorded into variables (`last_specter_event`, `last_specter_event_date`, `last_specter_payload`) and are saved to the app config so users see the last notice after restarts.
- **Faster, taller Variables viewer**: The Variables viewer height was increased and its population logic was optimized to be faster and more responsive (less freezing on navigation).
- **Debounced and coalesced updates**: Variable updates from events are throttled/queued so rapid event bursts won’t block the UI — the Variables page remains responsive even under heavy event load.
- **Cross-thread safety**: Variable updates are now dispatched via a queued Qt signal to ensure UI slots run in the GUI thread and avoid cross-thread QObject creation errors.

### Reliability

- **Non-blocking disconnect**: Disconnect requests are non-blocking — the OBSConnector thread performs the network disconnect, keeping the GUI responsive (fixes freezes when disconnecting).
- **Debounced scene precache**: Scene/source cache refreshes are scheduled and rate-limited instead of running immediately on every OBS event — reduces load on OBS and avoids MessageTimeout errors.
- **Worker loop scheduled precache**: The worker loop executes scheduled precaches with minimal interval between runs and guards against concurrent precaches.
- **Robust stats polling with improved error handling**: Query timeouts are handled gracefully and error states are logged with clearer messages.

### Bot & Twitch Integration

- **Save API Key flow**: `Validate` was renamed to `Save` and now persists the API key, updates the Twitch API object, and triggers a safe Channel Points refresh.
- **Improved logging**: Informational logs for Twitch fetches and Channel Points refreshes help you understand when and why calls happen.

## Bug Fixes 🐛

- Fixed UI freeze when disconnecting OBS WebSocket by moving disconnect logic into the connector thread.
- Stopped duplicate and repeated Twitch rewards fetches by adding debounce and scheduling logic on the Channel Points page.
- Fixed duplicate startup Channel Points refresh that could trigger two Twitch fetches on application launch; startup scheduling is now deduplicated and a unit test ensures a single refresh.
- Prevented overlapping scene/source precache runs and reduced the number of GetSceneList/GetSceneItemList calls to avoid obswebsocket MessageTimeouts.
- Fixed several edge-case crashes and added safer signal emission handling.
- Fixed Variables page freeze when opening and during rapid updates by scheduling refreshes, coalescing updates, and optimizing tree population.
- Corrected the Variables footer count so it reflects the items shown (defaults + variables) and the separate counters count.
- Prevented cross-thread QObject creation by emitting variable updates via a queued Qt signal (resolves "Cannot create children for a parent that is in a different thread" errors).
- Removed a small top-left version label to declutter the header and simplified the app title to "BotOfTheSpecter".

---

## Code Improvements & Maintenance 🔧

- **Refactor & modularization**: Introduced `theme.py` and reorganized UI layout for clarity and ease of future enhancements.
- **Better thread handling**: Worker threads (OBS connector, redemption handler, async thumbnail downloads) now avoid blocking the main thread and are safer on shutdown.
- **Variable change dispatch via Qt signals**: `VariableManager` now emits queued Qt signals for variable updates so UI listeners run in the GUI thread (improves cross-thread safety and avoids QObject creation errors).
- **Faster UI updates**: Variables page population and event coalescing were optimized to reduce freezes and improve responsiveness under load.
- **Improved logging & observability**: Added informative logs for Twitch fetches, thumbnail loads, Channel Points scheduling, and OBS precache scheduling.
- **Tests and safeguards**: Added debounce and guard logic around heavy network calls.
